Configuration (Setup)

Je suis le bot spécialisé dans l'automatisation du changement de logo, de nom, de bannière et d'image d'invitation de votre serveur Discord, sans nécessiter de configurations ou de lignes de code complexes.

Premiers pas

Pour commencer à utiliser le bot au maximum de ses capacités, lancez simplement la commande de configuration initiale.

  • /setup : Première commande à faire. Crée automatiquement la catégorie et les salons nécessaires (Logoto, log-logoto) pour un démarrage rapide.
NOTE IMPORTANTE : Le salon nommé log-logoto doit impérativement exister sur votre serveur pour assurer le suivi technique.

Le bot utilise ce salon pour envoyer ses messages techniques quotidiens de chargement (l-Loading, n-Loading, b-Loading ou i-Loading). Attention, ces messages ne sont envoyés que si un changement est effectivement programmé le jour même. S'il n'y a aucun événement, aucun log n'est posté. La commande /setup s'occupe de créer ce salon proprement pour vous.

Fonctionnement Général

Logoto surveille de manière autonome la présence de salons textuels spécifiques au sein de votre serveur Discord pour planifier et exécuter vos mises à jour esthétiques.

Planification horaire : Toutes les mises à jour et vérifications automatiques configurées sur votre serveur s'exécutent chaque jour précisément à 00h00 UTC.

Règles de priorité et limites

Pour garantir la stabilité du serveur et respecter les quotas de l'API Discord, Logoto applique des règles strictes de gestion quotidienne :

  1. Un seul changement par type et par jour : Le nom du serveur peut changer une fois par jour, le logo une fois, la bannière une fois, et le fond d'invitation une fois.
  2. Modifications simultanées : Il est tout à fait possible de changer le logo ET le nom (ou tout autre élément) le même jour. Les deux actions s'exécuteront à 00h00 UTC.
  3. Gestion des doublons (Priorité visuelle) : Si vous avez créé par erreur plusieurs salons pour le même élément le même jour (par exemple deux salons l-25-12 avec des logos différents), c'est le salon situé le plus haut dans la liste de vos catégories Discord qui prime et qui sera appliqué par le bot.

Les Commandes du Bot Logoto

Découvrez la liste complète des commandes intégrées disponibles directement avec le système de Slash Commands de Discord :

  • /setup : Crée les salons requis (Logoto, log-logoto) pour un démarrage rapide, indispensable pour le bon fonctionnement du bot.
  • /add-an-event type: jour: mois: : Crée un salon d'événement avec les options : [type] (obligatoire) pour le type d'événement (logo/name/banner/image invite), [day] (obligatoire) pour le jour de changement, et [month] (obligatoire) pour le mois de changement.
  • /about : Affiche les informations sur le bot, les liens d'invitation et les options de support.
  • /help : Affiche ce message d'aide (ou cette page de documentation).
  • /ping : Obtenez le ping du bot.

Ajouter un événement

Découvrez les détails de mise en place ci-dessous pour chaque type d'élément graphique ou textuel planifiable sur votre serveur Discord :

ATTENTION CONCERNANT LES IMAGES : Ne copiez jamais de liens d'images directement depuis un salon Discord (liens CDN Discord). Ces derniers possèdent désormais des clés de sécurité temporaires et expirent au bout de 24 à 48 heures. Si le lien expire avant la date de l'événement, Logoto ne pourra pas charger votre image. Référez-vous à la rubrique Liens d'images pour régler ce problème.

Configuration du Logo Automatique


  1. Créez le Salon de Planification :
    • Utilisez la commande slash /add-an-event type:Logo jour:[JOUR] mois:[MOIS] pour générer instantanément le salon.
    • Le salon créé par le bot suivra scrupuleusement le format : l-[JOUR]-[MOIS].
    • EXEMPLE : Pour programmer un logo pour Halloween (31 octobre) : l-31-10
  2. Préparez le Logo :
    • Obtenez un lien d'image permanent (via GitHub ou ImgBB) se terminant par un format valide (.png, .jpg, .jpeg ou .gif).
    • Taille recommandée par Discord : Un carré parfait de 512 × 512 pixels (les logos sont recadrés en cercle par l'application).
  3. Planifiez le Changement :
    • Modifiez le Sujet du Salon (Channel Topic) du salon textuel créé à l'étape 1 et collez l'URL permanente de l'image dedans.
  4. Résultat :
    • Le bot changera automatiquement la photo de profil du serveur au jour et au mois indiqués à 00h00 UTC. En cas de doublons de salons de logo, celui placé le plus haut l'emportera.

Configuration du Nom Automatique


  1. Créez le Salon de Planification :
    • Utilisez la commande slash /add-an-event type:Nom jour:[JOUR] mois:[MOIS] pour générer le salon.
    • Le salon créé par le bot suivra scrupuleusement le format : n-[JOUR]-[MOIS].
    • EXEMPLE : Pour planifier un changement de nom le 1er Janvier : n-1-1
  2. Planifiez le Changement :
    • Modifiez le Sujet du Salon (Channel Topic) du salon créé.
    • Écrivez textuellement le nouveau nom complet que devra prendre le serveur directement dans le sujet.
  3. Résultat :
    • Le bot appliquera automatiquement le nouveau nom au jour et au mois spécifiés à 00h00 UTC (limité à un seul changement de nom par jour, priorité au salon le plus haut).

Configuration de la Bannière Automatique


Note : Votre serveur doit posséder le niveau de Boost requis (Niveau 2+) pour appliquer des bannières personnalisées.

  1. Créez le Salon de Planification :
    • Utilisez la commande slash /add-an-event type:Bannière jour:[JOUR] mois:[MOIS] pour générer le salon.
    • Le salon créé par le bot suivra scrupuleusement le format : b-[JOUR]-[MOIS].
    • EXEMPLE : Pour planifier une bannière festive le 25 Décembre : b-25-12
  2. Préparez la Bannière :
    • Générez et copiez le lien direct permanent (URL) d'une image externe (.png, .jpg).
    • Taille recommandée par Discord : Des dimensions de 960 × 540 pixels avec un ratio de 16:9. Veillez à centrer les éléments importants pour éviter les recadrages.
  3. Planifiez le Changement :
    • Collez simplement ce lien permanent d'image dans le Sujet du Salon (Channel Topic).
  4. Résultat :
    • Logoto appliquera la bannière en arrière-plan au jour choisi à 00h00 UTC. Si plusieurs salons de bannières existent, celui situé le plus haut dans la liste sera traité.

Configuration du Fond d'Invitation Automatique


Note : Votre serveur doit posséder le niveau de Boost requis (Niveau 1+) pour appliquer une image de fond d'invitation personnalisée.

  1. Créez le Salon de Planification :
    • Utilisez la commande slash /add-an-event type:Fond d'invitation jour:[JOUR] mois:[MOIS] pour générer le salon.
    • Le salon créé par le bot suivra scrupuleusement le format : i-[JOUR]-[MOIS].
    • EXEMPLE : Pour un arrière-plan spécial le 14 Juillet : i-14-7
  2. Préparez l'Image de Fond :
    • Générez et copiez le lien direct permanent (URL) de l'image à intégrer.
    • Taille recommandée par Discord : Des dimensions de 1920 × 1080 pixels (ratio standard 16:9) pour un rendu net sur tous les écrans web et mobiles.
  3. Planifiez le Changement :
    • Collez le lien permanent de l'image de fond directement dans le Sujet du Salon (Channel Topic).
  4. Résultat :
    • Dès l'arrivée du jour choisi à 00h00 UTC, l'illustration viendra habiller vos fiches d'invitations web (priorité absolue au salon le plus haut en cas de doublons).

Obtenir un lien d'image persistant

Pour que Logoto puisse appliquer vos logos, bannières ou images d'invitation à l'avenir, l'URL que vous lui fournissez doit rester accessible de manière permanente. Comme les liens directs Discord expirent rapidement, vous devez utiliser une solution tierce d'hébergement permanent.

Choisissez l'une des méthodes recommandées ci-dessous pour générer un lien permanent valide :

Héberger via GitHub (Ultra-stable)


  1. Créez un dépôt public : Allez sur GitHub, cliquez sur New repository, donnez-lui un nom et laissez-le en Public.
  2. Important (Si le dépôt est vide) : Cochez la case "Add a README file" lors de la création.
    Pourquoi ? Si le dépôt est 100% vide, GitHub n'affiche que des lignes de code complexes et cache le bouton d'importation de fichiers.
  3. Téléversez vos images : Sur la page de votre dépôt désormais actif, cliquez sur le bouton "Add file" (en haut à droite) puis sélectionnez "Upload files". Glissez vos images (logo, bannière...) et cliquez sur le gros bouton vert "Commit changes" tout en bas pour valider.
  4. Obtenez l'URL brute (Raw) : Cliquez sur votre image dans la liste de GitHub pour l'ouvrir. Ensuite :
    • Cliquez sur le bouton "Raw" au-dessus de l'image (ou faites un clic droit sur l'image -> Ouvrir dans un nouvel onglet).
    • L'URL dans votre barre d'adresse doit commencer par : https://raw.githubusercontent.com/...
Astuce si vous ne trouvez pas le bouton "Raw" :
Prenez l'URL classique de la page de votre image sur GitHub, par exemple :
https://github.com/utilisateur/depot/blob/main/image.png
Et remplacez simplement manuellement le début pour obtenir l'URL finale directe :
https://raw.githubusercontent.com/utilisateur/depot/main/image.png

Hébergeurs Gratuits (ImgBB / Imgur)


  1. Rendez-vous sur un hébergeur d'images gratuit comme ImgBB ou Imgur.
  2. Téléversez votre image sur la plateforme sans y ajouter de date d'expiration.
  3. Une fois l'importation terminée, cherchez les options d'intégration et sélectionnez l'option "Lien direct" (Direct Link).
  4. Vérifiez l'URL obtenue : elle doit se terminer obligatoirement par l'extension du fichier (ex: .png, .jpg, .jpeg). Si le lien s'arrête sur du texte flou ou sans extension, Logoto ne pourra pas le lire.
  5. Copiez ce lien direct et ajoutez-le dans le sujet de votre salon de planification Discord.
